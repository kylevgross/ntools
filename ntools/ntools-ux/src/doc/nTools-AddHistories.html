Add Historiese

  • The AddHistories tool can be used to add histories to points based on haystack tagging.

Getting Started

  • To use this tool, simply copy it into your station, right click and select the Add History Extensions Action. Numeric Interval histories will be added to all numeric points tagged as a sensor, cmd, or sp. numeric points tagged as cmd will also get COV trends with a default change tolerance of 1. Boolean COV trends will be added to all boolean points tagged as a sensor, cmd, or sp. Enum COV trends will be added to all enum points tagged as a sensor, cmd, or sp.

Status

  • Current Status of the program.

Histories Last Added

  • Timestamp of the last time the program was successfully ran.

Number of Histories Generated

  • Count of how many history extensions were added.

History Name Format

  • The name format used on all history extensions except Numeric COV.

COV History Name Format

  • The name format used on Numeric COV history extensions.

Number of Numeric Interval Histories

  • This is the number of records to keep for each numeric interval history.

Collection Interval Numeric Interval Histories

  • This is how often the data is sampled for each numeric interval history.

Number of Numeric COV Histories

  • This is the number of records to keep for each numeric COV history.

Change Tolerance Numeric COV Histories

  • This is the default change tolerance value for each numeric COV history.

Number of Boolean COV Histories

  • This is the number of records to keep for each boolean COV history.

Number of Boolean Interval Histories

  • This is the number of records to keep for each boolean interval history. Since we are also doing COV we only need to keep a few of these and can space them out to once every 24 hours (see below).

Collection Interval Boolean Interval Histories

  • This is how often the data is sampled for each boolean interval history. Since we are aso doing COV we can spread these way out, just to keep recent data if they don't change often.

Number of Enum COV Histories

  • This is the number of records to keep for each enum COV history.

Number of Enum Interval Histories

  • This is the number of records to keep for each enum interval history. Since we are also doing COV we only need to keep a few of these and can space them out to once every 24 hours (see below).

Collection Interval Enum Interval Histories

  • This is how often the data is sampled for each enum interval history. Since we are aso doing COV we can spread these way out, just to keep recent data if they don't change often.
